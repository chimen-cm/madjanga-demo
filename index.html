10 décisions d'architecture documentées — contexte · alternatives rejetées · raison
ADR-01
Monorepo pnpm workspaces + Turborepo
Partage du moteur de calcul sans npm privé. Builds incrémentiels cachés. Un seul dépôt Git.
Rejeté : multi-repos séparés
ADR-02
Moteur calcul en package partagé @madjanga/engine
Doit fonctionner sans connexion sur le terrain camerounais. Même code = zéro désynchronisation frontend/backend.
Rejeté : calcul API uniquement
ADR-03
Hono plutôt qu'Express / NestJS
Léger (~12KB), TypeScript natif, edge-compatible. Express trop vieux. NestJS trop lourd pour une PME.
Rejeté : Express, NestJS, Fastify
ADR-04
Drizzle ORM plutôt que Prisma
Migrations SQL lisibles et versionnées. Support natif GENERATED ALWAYS. Requêtes typesafe proches du SQL brut.
Rejeté : Prisma, TypeORM
ADR-05
Supabase plutôt qu'AWS RDS + Cognito
PostgreSQL + Auth + Storage + Realtime en 1 service. ~25$/mois vs ~150$/mois AWS. SDK JS natif.
Rejeté : AWS RDS, Firebase
ADR-06
Zustand plutôt que Redux Toolkit
API simple, 1KB bundle, pas de slice/action/reducer. État DQE ligne par ligne sans re-render global.
Rejeté : Redux Toolkit, Context API
ADR-07
TanStack Table v8 plutôt qu'AG Grid
Open source MIT, virtualisation 10k+ lignes, édition in-place parfaite. AG Grid = licence commerciale coûteuse.
Rejeté : AG Grid Community
ADR-08
Colonnes GENERATED ALWAYS PostgreSQL
qte_cum et ht_ligne ne peuvent jamais être désynchronisés. Cohérence garantie SGBD, pas le code applicatif.
Rejeté : calcul applicatif uniquement
ADR-09
audit_event append-only via REVOKE SQL
Exigence contractuelle CCAP. REVOKE DELETE/UPDATE = impossible à contourner même par un bug applicatif.
Rejeté : soft-delete, log externe
ADR-10
Workbox + Dexie.js pour l'offline
Standards Google Service Workers + API Promise propre sur IndexedDB. Matures, bien documentés, stratégies prêtes à l'emploi.
Rejeté : SW custom, PouchDB
